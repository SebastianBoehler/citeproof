# Assertion Status Role Lens Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent high-overlap evidence about hypotheses, future work, design intent, role/provenance swaps, or ties from producing false `supported` labels.

**Architecture:** Keep assertion status and role/provenance binding as separate deterministic fact lenses. Wire hard role conflicts and soft assertion-status tensions through `inspect_facts`, then extend comparison parsing so neutral tie evidence cannot satisfy stronger outperformance claims.

**Tech Stack:** Python 3.11+, pytest, ruff, existing CiteProof fact-lens architecture.

---

## File Structure

- Create `src/citeproof/assertion_lens.py`: future/hypothesis/design-intent tensions.
- Create `src/citeproof/role_lens.py`: controlled role/provenance binding conflicts.
- Create `tests/test_assertion_lens.py`: direct status-cue tests and false-positive guards.
- Create `tests/test_role_lens.py`: direct role/provenance conflict tests and supported boundaries.
- Create `tests/test_entailment_assertion_roles.py`: end-to-end adjudication labels.
- Modify `src/citeproof/fact_lenses.py`: include both new lenses.
- Modify `src/citeproof/adjudicator.py`: map role conflicts to `entity_conflict`.
- Modify `src/citeproof/comparison_lens.py` and `tests/test_comparison_lens.py`: neutral tie relation tension.
- Modify `examples/edge_cases/claim_support.jsonl`, `tests/test_eval_runner.py`, and `docs/evaluation.md`: benchmark rows and score docs.

from citeproof.models import FailureMode, Label

    judgment = adjudicate_evidence(
        "The method improves calibration.",
        "Future work will test whether the method improves calibration.",
    )
    assert judgment.label == Label.PARTIALLY_SUPPORTED
    assert judgment.failure_mode == FailureMode.SCOPE_OVERSTATEMENT


def test_role_binding_blocks_support() -> None:
    judgment = adjudicate_evidence(
        "GPT-4 generated the dataset annotations.",
